# 🚀 快速开始指南\n\n## 方案A：使用Python初始化脚本（推荐）⭐\n\n### 步骤1：克隆仓库\n```bash\ngit clone https://github.com/emmm-up/automated-test-framework.git\ncd automated-test-framework\n```\n\n### 步骤2：运行初始化脚本\n\n#### macOS/Linux\n```bash\npython3 init-project.py\n```\n\n#### Windows\n```bash\npython init-project.py\n```\n\n脚本将自动：\n- ✅ 创建完整的项目目录结构\n- ✅ 生成所有配置和代码文件\n- ✅ 创建Python虚拟环境\n- ✅ 初始化Git仓库\n- ✅ 生成详细的操作日志\n\n### 步骤3：激活虚拟环境\n\n#### macOS/Linux\n```bash\nsource venv/bin/activate\n```\n\n#### Windows\n```bash\nvenv\\Scripts\\activate\n```\n\n### 步骤4：安装依赖\n```bash\npip install -r requirements.txt\n```\n\n### 步骤5：配置环境变量\n```bash\ncp .env.example .env\n# 编辑 .env 文件，填入你的配置\n```\n\n### 步骤6：运行测试\n```bash\n# 运行所有测试\npytest -v\n\n# 运行特定测试文件\npytest tests/api/test_example_api.py -v\n\n# 生成Allure报告\npytest --alluredir=allure-results\nallure serve allure-results\n```\n\n---\n\n## 方案B：使用Docker（容器化部署）\n\n### 步骤1：克隆仓库\n```bash\ngit clone https://github.com/emmm-up/automated-test-framework.git\ncd automated-test-framework\n```\n\n### 步骤2：使用docker-compose运行\n```bash\ndocker-compose -f docker/docker-compose.yml up\n```\n\n### 步骤3：构建自定义镜像\n```bash\n# 构建镜像\ndocker build -f docker/Dockerfile -t auto-test-framework:latest .\n\n# 运行容器\ndocker run -v $(pwd):/app auto-test-framework:latest\n```\n\n---\n\n## 常见问题\n\n### Q1: Python版本要求?\nA: Python 3.9+\n\n### Q2: 如何查看初始化日志？\nA: 查看 `init-log.txt` 文件\n\n### Q3: 如何重新初始化项目？\nA: 删除 `venv` 目录后重新运行脚本\n\n### Q4: Windows上执行脚本失败？\nA: 尝试使用 `init-project.bat` 或确保已安装Python\n\n---\n\n## 项目结构\n\n```\nautomated-test-framework/\n├── framework/                 # 核心框架\n│   ├── config/               # 配置管理\n│   ├── base/                 # API客户端基类\n│   └── utils/                # 工具函数\n├── tests/                    # 测试用例\n│   └── api/                  # API测试\n├── docker/                   # Docker配置\n├── .github/workflows/        # CI/CD配置\n├── docs/                     # 文档\n├── logs/                     # 日志输出\n├── requirements.txt          # Python依赖\n├── pytest.ini               # Pytest配置\n├── setup.py                 # 项目配置\n├── .env.example             # 环境变量模板\n└── init-project.py          # 初始化脚本\n```\n\n---\n\n## 下一步\n\n1. 📖 查看 `docs/` 目录了解详细技术方案\n2. 📝 查看 `tests/api/test_example_api.py` 了解如何编写测试\n3. 🔧 修改 `.env` 配置测试环境\n4. ✨ 开始编写你的测试用例！\n\n---\n\n## 需要帮助？\n\n- 📚 技术选型：`docs/技术选型方案.md`\n- 🛠️ 部署指南：`docs/环境部署指南.md`\n- 🚀 CI/CD实践：`docs/CI_CD集成实践.md`\n- 📋 任务分解：`docs/项目任务分解.md`\n\n---\n\n**祝你使用愉快！🎉**\n
